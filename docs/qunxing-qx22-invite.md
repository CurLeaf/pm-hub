# 群兴 QX-22：分中心邀请普通合伙人流程（规格）

与看板 **QX-12**（分中心成员/员工档案审核）并存；本规格仅约束 **普通合伙人** 邀请链路。

## 范围

企业向**普通合伙人**发出邀请，合伙人通过链接完成资料填报与审核；与中台/分中心审核能力衔接。

## 流程拆解

1. **被邀请门槛**  
   普通合伙人被邀请时须填写：**个人信息**、**银行卡信息**、上传**身份证**、**征信报告**；**全部经审核通过**后，邀请才算成功。

2. **资料完整性（合伙人侧）**  
   填报齐全后，**邀请管理 → 邀请信息列表** 中该条 **状态** 为 **待审核**。  
   - **个人信息**（示例字段）：姓名、性别、出生日期、国籍、种族、手机号码、邮箱、通讯地址、学历、职位。  
   - **银行卡信息**：卡号、开户地址。  
   - **身份证**：正反面；支持 **图片与 PDF**。  
   - **征信报告**：上传。  
   - **说明**：分中心向中台提交申请时需填写 **公司信息** 与 **公司银行对公账户**；**普通合伙人侧使用私户即可**。

3. **审核操作（分中心侧）**  
   在 **分中心审核合伙人** 区域，对合伙人已填信息可 **同意**（蓝色）或 **拒绝**（红色）。

4. **打回后**  
   邀请被拒后，进入 **补充信息** 流程，可再次提交审核。

5. **端到端链路**  
   企业发邀请链接 → 合伙人打开链接填报 → 企业审核；合伙人侧链接在审核中展示 **待审核** → **通过** 则加入企业；**未通过** 则链接侧展示不通过，须 **补充资料后重新提交**；**全流程带时间限制**。

## 验收

- 普通合伙人从被邀请到 **填完个人信息、银行卡、身份证、征信报告，并审核通过** 的链路可完整走通。  
- **任一环节不可跳过**（含必填项、审核、补件、时效内完成等，以实现为准）。
